## 觀念題與名詞解釋:精簡 摘要說明底下觀念 [請說明重要性][每題兩分]
```
1.最小權限原則 (Principle of least privilege)
  給予使用者所需最小權限。
2.原型（prototype）|概念驗證（poc, proof of concept）
3.單元測試（unit test）
  對單個程式、函式進行正確性的測試。
4.驗收測試（acceptance test）
  終端使用者確認新系統是否符合需求。
5.先導轉換（pilot conversion）
  選一個部門是試用新系統，確認符合需求後，其他部門在更換。
6.Von Neumann architecture
  是一種將程式指令記憶體和資料記憶體合併在一起的電腦設計概念架構。
7.記憶體階層(Memory hierarchy)
  暫存器
  快取:
    L1:xxK
    L2:xM
    L3:xxM
  RAM
  Hard Disk
8.機器週期
    1.擷取:擷取要計算的資料
    2.解碼:將計算指令解碼
    3.執行:進行資料計算
    4.儲存:將結果存入記憶體
9.RAID[RAID 0 / RAID 1/RAID 0+1]、NAS、SAN
   RAID
    利用虛擬化儲存技術把多個硬碟組合起來。
    RAID 0
      將2個以上的硬碟並聯起來，讀寫能並列處理。速度最快
    RAID 1
      2個以上硬碟互相鏡像，資料損失時可以互補。容量只有一半。
    RAID 0+1     
   
10.近距離無線通訊（Near-field communication，NFC）
    使用無線電連接技術，讓兩台裝置能夠在幾公分內進行通訊。
    行動支付
```
# 問答題:[每題五分]
```
1. 說明檔案處理系統與資料庫 系統 這兩種方法的差異 [要說明優缺點]
    檔案處理系統:
      每個部門都有自己一組的檔案。
     優:簡單
     缺:資料重複性高、資料被隔離
    資料庫:
      讓使用者共享一個資料庫。
     優:降低資料重複性、資料共享
     缺:較複雜、容易被入侵
2. 說明大數據 的 3V 特性與 5V特性
   Volume 數量
    資料量巨大。
   Velocity  速度
    資料產生速度快、處理速度快。
   Variety 多樣性
     資料呈現的多種格式。包刮結構化、半結構化、非結構化。
   Value 價值
     價值密度低
   Veracity 真實性
     數據的正確性。
3. 說明系統開發生命週期（system development life cycle）有哪些階段?每階段要做甚麼事?
    
  
4. 請舉 程式語言 例說明編譯器 （compiler）與直譯器（interpiler）的差別
    編譯器:一次將寫好的原始碼轉成機器語言。 C、C++
    直譯器:將一行一行的原始碼轉換成機器語言並執行。Python、Ruby
```
